# Лабораторная работа №1  
## Освоение базового ML-пайплайна на табличных данных

---

## Описание

В рамках работы реализован полный цикл машинного обучения на табличных данных: от разведочного анализа до обучения моделей и оценки их качества.

Цель - спрогнозировать:
- **CGPA** (успеваемость) — задача регрессии  
- **Depression** (риск депрессии) — задача классификации  

---

## Этапы работы

### 1. Разведочный анализ данных (EDA)
- анализ структуры данных  
- проверка пропусков и дубликатов  
- визуализация распределений  
- поиск выбросов  

---

### 2. Предобработка данных
- удаление неинформативных признаков  
- кодирование категориальных признаков (One-Hot Encoding)  
- создание новых признаков (feature engineering):  
  - `Sleep_Study_Ratio`  
  - `Total_Load`  
  - `Health_Index`  
- масштабирование признаков (StandardScaler)  
- учёт дисбаланса классов (`class_weight='balanced'`)  

---

### 3. Разделение данных
Данные были разделены на:
- **Train** — 60%  
- **Validation** — 20%  
- **Test** — 20%  

Без утечки данных (масштабирование после split).

---

### 4. Модели

#### Linear Regression
Используется для прогнозирования CGPA.

#### Logistic Regression
Используется для определения риска депрессии.

---

### 5. Метрики

#### Регрессия:
- MSE  
- R²  

#### Классификация:
- Accuracy  
- Precision  
- Recall  
- F1-score  
- ROC-AUC  

---

### 6. Проверка переобучения
Проводилась путём сравнения метрик на:
- обучающей выборке  
- валидационной выборке  

Разница незначительна → переобучения нет.

---

### 7. Эксперименты
Проведены эксперименты с:
- learning rate  
- числом эпох  

Использовался `SGDClassifier`.

---

## Используемые технологии

- Python  
- Pandas  
- NumPy  
- Matplotlib / Seaborn  
- Scikit-learn  

---

## Выводы

- Предобработка и feature engineering существенно улучшили качество моделей  
- Учёт дисбаланса классов оказался критически важным  
- Logistic Regression показала способность выявлять случаи депрессии  
- Linear Regression адекватно прогнозирует CGPA  
- Переобучение отсутствует  
- Гиперпараметры (learning rate, epochs) влияют на качество модели  

---

## Трудности

- утечка данных при масштабировании  
- дисбаланс классов (F1 = 0 на начальном этапе)  
- настройка SGDClassifier (и понимание что это такое)
